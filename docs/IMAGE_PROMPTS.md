# Image Prompts

This document stores prompt guidance for future README and website visuals. The goal is to keep visuals consistent with the product narrative: structured failure research, not meme graveyard aesthetics.

## Global art direction

- Tone: analytical, sharp, modern, high-signal
- Avoid: cartoon cemeteries, joke visuals, Halloween motifs, exaggerated tombstone imagery
- Preferred visual language: research interface, network graph, structured taxonomy, intelligence dashboard, annotated case flows
- Color direction: deep navy, slate, muted electric blue, restrained signal accents
- Composition: editorial product concept art, minimal noise, clear focus

## 1. Hero concept

Planned asset: `docs/assets/readme-hero-concept.png`

Prompt:

> Create a cinematic editorial product illustration for an open-source failure intelligence platform. Show a dark research workspace with layered structured case cards, timelines, taxonomy nodes, funding loss charts, and linked failure patterns. The scene should feel analytical and credible, like a startup research operating system, not a horror or graveyard joke. Use deep navy and slate with restrained electric blue highlights. Add subtle data-grid texture and soft directional lighting. High-end SaaS illustration quality, clean composition, wide landscape format.

## 2. User value map

Planned asset: `docs/assets/readme-user-value-map.png`

Prompt:

> Create a product strategy diagram illustration showing how founders, investors, and researchers use the same structured startup failure dataset differently. Visualize one shared dataset in the center branching into three usage modes: pattern recognition, diligence support, and failure research. Minimal editorial infographic style, premium documentation visual, navy and blue palette, clean labels, subtle depth, no clutter.

## 3. Product architecture

Planned asset: `docs/assets/readme-product-architecture.png`

Prompt:

> Create a clean system architecture illustration for a startup failure intelligence platform. Show three layers: public research surface, admin and operations surface, and platform/data layer. Include structured case records, research hub, AI copilot, saved views, admin review, ingestion pipeline, embeddings, taxonomy normalization, and contracts. Make it feel like a polished engineering diagram with editorial visual quality, not a raw technical block diagram. Deep navy background, blue accents, crisp labels.

## 4. Case lifecycle

Planned asset: `docs/assets/readme-case-lifecycle.png`

Prompt:

> Create a lifecycle infographic showing the flow from source URL to source snapshot, draft case, evidence review, structured signal extraction, published case, search index, and research outputs. The style should feel like a premium product operations explainer for an intelligence platform. Use linked cards, arrows, step markers, and restrained motion cues. Clean, readable, minimal, dark professional palette.

## Placement guidance

- README hero: hero concept first
- README middle section: product architecture or lifecycle visual
- Docs or future landing page: user value map and lifecycle visual

## Output guidance

- Prefer 1600px+ wide PNG
- Keep enough negative space for cropping into social cards if needed
- Avoid embedding lots of small text in the generated image
